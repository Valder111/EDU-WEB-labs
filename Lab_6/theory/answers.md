
# Теоретические вопросы по современной веб-разработке

## 1. В чём разница между ECMA-262, ECMAScript и JavaScript?

- **ECMA-262** — это официальный стандарт, разработанный и поддерживаемый организацией Ecma International. Он формально описывает синтаксис, семантику и поведение языка программирования.
- **ECMAScript** — это название языка программирования, определённого в стандарте ECMA-262.
- **JavaScript** — это торговая марка и одна из наиболее известных реализаций стандарта ECMAScript. Используется в веб-браузерах и серверных средах (например, Node.js).

Таким образом, ECMAScript — это спецификация, а JavaScript — её практическая реализация.

## 2. Что такое полифилл, транспайлинг, Babel, SWC и как они связаны между собой?

- **Транспайлинг** — процесс преобразования исходного кода из одной версии языка в другую, обычно из современной — в более старую, совместимую с целевыми средами выполнения.
- **Полифилл (polyfill)** — программный код, который эмулирует функциональность, отсутствующую в старых версиях среды выполнения (например, реализация `Promise` для Internet Explorer).
- **Babel** — популярный транспайлер для JavaScript, позволяющий использовать современные возможности языка (ES2015+) в старых браузерах. Может интегрироваться с библиотеками полифиллов (например, `core-js`).
- **SWC (Speedy Web Compiler)** — высокопроизводительный транспайлер, написанный на Rust. Выполняет те же задачи, что и Babel, но значительно быстрее.

**Связь**: Babel и SWC — инструменты транспайлинга; полифиллы дополняют их, обеспечивая совместимость не только синтаксиса, но и API среды выполнения.

## 3. Что такое bundler, webpack, vite и как они связаны между собой?

- **Bundler (сборщик)** — инструмент, объединяющий множество модулей, стилей, изображений и других ресурсов в один или несколько файлов (бандлов), оптимизированных для загрузки в браузере.
- **Webpack** — гибкий и мощный bundler, поддерживающий сложные сценарии сборки: код-сплиттинг, ленивая загрузка, транспайлинг, минификация и работа с assets. Требует конфигурации и может быть медленным при запуске.
- **Vite** — современный инструмент сборки, использующий нативные ES-модули в режиме разработки, что обеспечивает мгновенный запуск. Для продакшн-сборки использует Rollup. Отличается простотой настройки и высокой скоростью.

**Связь**: Webpack и Vite — это конкретные реализации концепции bundler’а, решающие задачу подготовки фронтенд-проекта к развёртыванию.

## 4. Что такое WebSocket и чем он отличается от Server-Sent Events (SSE)?

| Характеристика                                                   | **WebSocket**                                                                | **Server-Sent Events (SSE)**                                        |
| ------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| Направление связи                                              | Двусторонняя (full-duplex)                                             | Односторонняя (сервер → клиент)                 |
| Протокол                                                               | Независимый (`ws://`, `wss://`)                                     | Основан на HTTP                                                  |
| Поддержка данных                                                | Текст и бинарные данные                                        | Только текст (UTF-8)                                           |
| Автоматическое восстановление соединения | Нет (требуется ручная реализация)                      | Да                                                                      |
| Совместимость с браузерами                             | Широкая (все современные)                                     | Все, кроме Internet Explorer                                      |
| Типичное применение                                          | Чаты, онлайн-игры, совместное редактирование | Ленты новостей, уведомления, мониторинг |

Ключевое различие: WebSocket обеспечивает **полноценный двусторонний канал связи**, тогда как SSE — это **поток данных от сервера к клиенту** через обычное HTTP-соединение.

## 5. TypeScript: преимущества и недостатки по сравнению с языками со встроенной статической типизацией

### Преимущества TypeScript:

- Постепенная (опциональная) типизация: можно интегрировать в существующие JavaScript-проекты.
- Полная совместимость с JavaScript: любой корректный JS-код является валидным TypeScript.
- Богатая система типов: интерфейсы, generics, union/intersection типы, type guards.
- Раннее выявление ошибок на этапе разработки (а не выполнения).
- Отличная поддержка в редакторах (особенно VS Code) и экосистеме (React, Vue, Angular и др.).

### Недостатки по сравнению с «нативными» типизированными языками (например, Java, C#, Rust):

- Типы существуют **только на этапе компиляции** и не влияют на выполнение программы.
- Отсутствие **runtime-типов** → невозможность выполнять надёжную динамическую проверку типов без дополнительного кода.
- Сложность при работе с динамическими или слабо структурированными данными (например, произвольные JSON-ответы от API).
- Требует дополнительного этапа сборки (компиляции в JavaScript), что усложняет простые проекты.

**Вывод**: TypeScript значительно повышает надёжность и поддерживаемость крупных JavaScript-приложений, но не предоставляет всех гарантий, характерных для языков со статической типизацией «из коробки».
